XHTML3プロジェクトへ参加しよう

あなたはHTMLの仕様を知っていますか？HTMLの仕様はWHATWGによってHTML Living Standardと呼ばれています。
https://html.spec.whatwg.org/multipage/
日本語翻訳:https://momdo.github.io/html/
その中、私はHTMLをXMLで再定義したXHTML3を作りたいと思っていて、まだ作り始めですがXHTML3のXSDをすでに用意しています。
https://github.com/inomoto-hironobu/framexs/blob/master/xsd/xhtml3.xsd

XHTML3の仕様を考えているのは上記で述べたFramexsとの関係もありますが、なによりHTMLとは何か、をよく考えるためです。
XHTML3計画に参加することはHTMLをより深く知ることにつながります。

そして、XHTMLはHTMLとXMLの両方の性質を持っています。つまりXHTML3計画を通してXHTMLを学べば、HTMLとXMLの両方の知識が手に入ります。 XSLT3.0などの最新技術によって、XMLは今後のWeb開発において重要な要素を担うことになります。

FileHTML拡張プロジェクト

HTMLはfileプロトコルでアクセスできます。例えば、
file:///C:/html/foo.html というURLでローカルのHTMLを読むことができます。
このようにアクセスされるHTMLを、FileHTMLと呼ぶことにして話を勧めますが、このFileHTMLをブラウザで読み込むときは、様々な制限があります。 例えばJavaScriptで他のリソースにアクセスすることができません。

 response.json())
  .then((data) => console.log(data));]]>
とすれば、 
Access to fetch at 'file:///C:/html/foo.json' from origin 'null' has been blocked by CORS policy: Cross origin requests are only supported for protocol schemes: http, data, isolated-app, chrome-extension, chrome, https, chrome-untrusted.

となります。これを変えてみましょう。仮に、

]]>

とすれば、

 response.json())
  .then((data) => console.log(data));]]>

と、リソースにアクセスできるようにするのです。
こうすると、例えばOneDrive、DropBoxなどのクラウドストレージ、あるいはNASなどのイントラストレージで有効活用すると、事務作業が大いにはかどるはずです。
現状、クラウドストレージは、GASによってGoogleドライブが他より多く使われている傾向になっています。
Chromeが多く使われていることも考えると、Googleの天下ですね。
ここでもし、MicrosoftがFirefoxを支援して、FileHTML拡張機能を持つようになったなら、どうなるでしょう。
とてもワクワクすることが起きると思います。
